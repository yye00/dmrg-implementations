================================================================================
ROOT CAUSE ANALYSIS
================================================================================

**Why is it so slow?**

The bottleneck is Phase 2: Parallel local micro-steps
- Each site performs a local eigenvalue problem
- For L=20, that's 19 two-site problems (or 20 one-site problems)
- Each eigensolver call takes ~2-3 seconds
- With L sites, total time ≈ L * 2-3 seconds

**Why does it scale as O(L²)?**

1. Number of sites: O(L)
2. Each site's effective Hamiltonian involves contracting environment tensors
3. Environment contraction cost: O(L * χ³) where χ is bond dimension
4. Total: O(L²χ³) per sweep

For large L, the tensor contractions dominate.

================================================================================
OPTIMIZATION OPPORTUNITIES
================================================================================

**Potential Speedups:**

1. **Parallelize local micro-steps** (ALREADY DESIGNED FOR THIS!)
   - With np=4: Could reduce time by ~4x
   - L=20 sweep: 66s → ~17s
   - BUT: MPI overhead may reduce efficiency

2. **Cache environment tensors**
   - Reuse left/right environments across sites
   - Could save ~50% of contractions
   - L=20 sweep: 66s → ~33s

3. **Use faster eigensolver**
   - Current: scipy.sparse.linalg.eigsh
   - Alternative: ARPACK with better settings
   - Potential: 20-30% speedup

4. **Reduce bond dimension**
   - bond_dim=32 instead of 50 for L=20
   - Slight accuracy loss but much faster
   - Time reduction: ~50%

5. **Early stopping**
   - Stop when ΔE < tolerance
   - Most tests would complete in 3-5 sweeps instead of 20
   - Time reduction: ~75%
