<project_specification>
  <project_name>Additive Two-Level Parallel DMRG (A2DMRG)</project_name>

  <overview>
    Implement the Additive Two-Level Parallel DMRG algorithm from Grigori & Hassan 
    (arXiv:2505.23429). This algorithm achieves parallel speedup through an additive 
    Schwarz approach: all TT-cores are updated independently in parallel, then combined
    via a coarse-space minimization step.
    
    Key Innovation: Unlike sequential DMRG that updates one site at a time, A2DMRG
    performs independent local micro-steps for ALL sites simultaneously, then finds
    an optimal linear combination of all updated states via coarse-space minimization.
    This is directly inspired by domain decomposition methods from numerical PDEs.
    
    The implementation must:
    1. Achieve near-linear scaling for 2-8 processors
    2. Converge to machine precision (match serial DMRG within 1e-10)
    3. Work with complex128 (for Josephson junction problems)
    4. Use mpi4py for parallelization, quimb for tensor operations
    5. NOT use quimb's native DMRG inside A2DMRG (only for validation)
    6. Generate scalability reports comparing np=1,2,4,8
  </overview>

  <references>
    <paper>Grigori & Hassan, "An additive two-level parallel variant of the DMRG 
           algorithm with coarse-space correction", arXiv:2505.23429v2 (2025)</paper>

  <!-- A2DMRG ALGORITHM - COMPLETELY DIFFERENT FROM PDMRG -->
  <algorithm_overview>
    <key_difference>
      A2DMRG is fundamentally different from PDMRG (Stoudenmire-White):
      - NO real-space partitioning
      - NO V = Λ⁻¹ trick
      - NO neighbor exchanges at boundaries
      - NO staggered sweep directions
      
      Instead:
      - ALL sites update in parallel (additive Schwarz)
      - Coarse-space minimization combines results
      - Truncated SVD for compression (standard, not "accurate")
    </key_difference>

    <phase name="0" title="Initialization">
      <description>
        Start with a left-orthogonal tensor train decomposition U^(0).
        Can use random initialization or a few serial DMRG warmup sweeps.
        Run initialization on rank 0, then broadcast to all ranks.
      </description>
      <outputs>Left-orthogonal MPS U^(0) = (U₁, U₂, ..., U_d)</outputs>
      <left_orthogonal_property>
        A TT decomposition is left-orthogonal if for all i &lt; d:
        Σₛ Uᵢ(:,s,:)ᵀ Uᵢ(:,s,:) = I  (each slice is orthonormal)
      </left_orthogonal_property>
    </phase>

    <phase name="1" title="Prepare Orthogonal Decompositions">
      <description>
        From left-orthogonal U^(n,d), construct d different i-orthogonal 
        decompositions U^(n,i) for i = 1, 2, ..., d.
        
        An i-orthogonal decomposition has:
        - Sites 1 to i-1: left-orthogonal
        - Site i: the "orthogonality center" (contains the norm)
        - Sites i+1 to d: right-orthogonal
      </description>
      <procedure>
        Starting from U^(n,d) (left-orthogonal), sweep right-to-left:
        For i = d, d-1, ..., 2:
          1. LQ decomposition: W_i^(1) = L_i · Q_i
          2. Update: W_{i-1}^(n) = U_{i-1}^(n) · L_i
          3. Set: V_i^(n) = Q_i (right-orthogonal)
        Result: d decompositions U^(n,i) ready for parallel local steps
      </procedure>
      <parallelization>
        This preparation CAN be done sequentially on rank 0 OR distributed.
        The decompositions are small tensors, so sequential is often fine.
        Broadcast the d decompositions to all processors.
      </parallelization>
    </phase>

    <phase name="2" title="Parallel Local DMRG Micro-Steps (CORE PARALLEL STEP)">
      <description>
        This is where parallelism happens. Each processor handles one or more
        sites and performs independent DMRG micro-steps WITHOUT communication.
        
        For one-site DMRG: compute Vᵢ = Sᵢ(U^(n,i)) for each site i
        For two-site DMRG: compute W_{i,i+1} = S_{i,i+1}(U^(n,i)) for each bond
      </description>
      <distribution>
        With P processors and d sites:
        - sites_per_proc = d // P
        - Processor p handles sites [p * sites_per_proc, (p+1) * sites_per_proc)
        - Remainder sites distributed to first (d % P) processors
      </distribution>
      <one_site_micro_step>
        Given i-orthogonal decomposition U^(n,i):
        1. Build left environment L_{i-1} from sites 1..i-1
        2. Build right environment R_{i+1} from sites i+1..d
        3. Build effective Hamiltonian H_eff at site i
        4. Solve local eigenvalue problem: H_eff v = λ v
        5. Set Vᵢ = v.reshape(r_{i-1}, n_i, r_i)
        6. Construct updated TT decomposition Ũ^(n+1,i) using Vᵢ
      </one_site_micro_step>
      <two_site_micro_step>
        Given i-orthogonal decomposition U^(n,i):
        1. Build left environment L_{i-1}
        2. Build right environment R_{i+2}
        3. Build effective Hamiltonian H_eff for sites i, i+1
        4. Solve: H_eff w = λ w
        5. Reshape w to W_{i,i+1} with shape (r_{i-1}, n_i, n_{i+1}, r_{i+1})
        6. Truncated SVD to split W_{i,i+1} back to two cores
        7. Construct updated TT decomposition Ũ^(n+1,i)
      </two_site_micro_step>
      <no_communication>
        CRITICAL: During this phase, there is NO MPI communication.
        Each processor works completely independently on its assigned sites.
        This is what makes A2DMRG embarrassingly parallel.
      </no_communication>
    </phase>

    <phase name="3" title="Coarse-Space Minimization (CRITICAL STEP)">
      <description>
        After all processors complete their local micro-steps, we have d+1 
        candidate tensors:
        - Y^(0) = τ(U^(n,d))  (the original state)
        - Y^(j) = τ(Ũ^(n+1,j)) for j = 1, ..., d (from local micro-steps)
        
        Find optimal linear combination that minimizes the energy:
        
        For one-site: min_{c₀,...,c_d} J(Σⱼ cⱼ Y^(j))
        For two-site: min_{d₀,...,d_{d-1}} J(Σⱼ dⱼ Y^(j))
      </description>
      <coarse_space_problem>
        This reduces to a small (d+1) × (d+1) generalized eigenvalue problem:
        
        Find c* such that: H_coarse c* = λ S_coarse c*
        
        Where:
        - H_coarse[i,j] = ⟨Y^(i), A Y^(j)⟩  (Hamiltonian matrix)
        - S_coarse[i,j] = ⟨Y^(i), Y^(j)⟩    (overlap/mass matrix)
        
        The optimal combination is: Ỹ = Σⱼ c*ⱼ Y^(j)
      </coarse_space_problem>
      <mpi_implementation>
        1. Each processor computes its local contributions to H_coarse and S_coarse
        2. Allreduce to sum contributions from all processors
        3. Rank 0 solves the small eigenvalue problem (d+1 dimensional)
        4. Broadcast coefficients c* to all processors
      </mpi_implementation>
      <numerical_considerations>
        The mass matrix S_coarse may be ill-conditioned.
        Use SVD-based pseudoinverse or regularization:
        
        S = U Σ V^H
        S^(-1/2) = U Σ^(-1/2) V^H  (truncate small singular values < 1e-10)
        
        Transform to standard eigenvalue problem:
        S^(-1/2) H S^(-1/2) c̃ = λ c̃
        Then c* = S^(-1/2) c̃
      </numerical_considerations>
    </phase>

    <phase name="4" title="Compression/Projection">
      <description>
        The optimal linear combination Ỹ = Σⱼ c*ⱼ Y^(j) may have increased
        TT-ranks. Project back to the target rank manifold T_r.
      </description>
      <truncated_svd>
        Use TT-SVD algorithm (standard, NOT recursive accurate SVD):
        1. Start from leftmost core
        2. For i = 1 to d-1:
           a. Reshape: M = U_i.reshape(r_{i-1} * n_i, -1)
           b. SVD: M = P Σ Q^H
           c. Truncate: keep top r_i singular values (or use tolerance 1e-6)
           d. Set: U_i = P[:, :r_i].reshape(r_{i-1}, n_i, r_i)
           e. Propagate: U_{i+1} = (Σ[:r_i] @ Q[:r_i, :]) @ U_{i+1}
        3. Result: compressed left-orthogonal TT with bounded ranks
      </truncated_svd>
      <rank_selection>
        Option 1: Fixed maximum rank r_max
        Option 2: Adaptive with truncation tolerance ε = 1e-6
        Option 3: Dynamic based on singular value decay
        
        Paper uses: tolerance = 1e-6 for truncated SVD
      </rank_selection>
    </phase>

    <phase name="5" title="Convergence Check and Iteration">
      <description>
        After compression, we have U^(n+1) with controlled ranks.
        Check convergence and repeat if needed.
      </description>
      <convergence_criteria>
        1. Energy difference: |E^(n+1) - E^(n)| &lt; tol (typically 1e-8 to 1e-10)
        2. Relative energy: |E^(n+1) - E^(n)| / |E^(n)| &lt; rel_tol
        3. Maximum iterations: n &lt; max_sweeps
      </convergence_criteria>
      <energy_computation>
        E^(n+1) = ⟨τ(U^(n+1)), A τ(U^(n+1))⟩ / ∥τ(U^(n+1))∥²
        
        For efficiency, compute during coarse-space step (already have H and S).
      </energy_computation>
    </phase>
  </algorithm_overview>

  <numerical_methods>
    <truncated_svd title="Standard Truncated SVD">
      <description>
        A2DMRG uses standard truncated SVD (NOT the recursive accurate SVD from PDMRG).
        This is because A2DMRG does not invert singular values.
      </description>
      <tolerance>
        Paper uses tolerance = 1e-6 for truncated SVD.
        Truncate singular values σ_i where σ_i / σ_1 &lt; 1e-6.
      </tolerance>
      <implementation>
        def truncated_svd(M, max_rank=None, tol=1e-6):
            U, S, Vh = np.linalg.svd(M, full_matrices=False)
            
            # Truncate by tolerance
            if tol > 0:
                cutoff = S[0] * tol
                keep = np.sum(S > cutoff)
            else:
                keep = len(S)
            
            # Truncate by max rank
            if max_rank is not None:
                keep = min(keep, max_rank)
            
            return U[:, :keep], S[:keep], Vh[:keep, :]
      </implementation>
    </truncated_svd>

    <eigensolver title="Lanczos/Davidson for Effective Hamiltonian">
      <effective_hamiltonian>
        H_eff acts on one-site tensor (χ_L, d, χ_R) or two-site (χ_L, d, d, χ_R).
        Built from: L_env, W_i, (W_{i+1}), R_env
        
        NEVER form H_eff as explicit matrix!
        Use scipy.sparse.linalg.LinearOperator for matrix-free application.
      </effective_hamiltonian>
      
      <usage>
        from scipy.sparse.linalg import eigsh, LinearOperator
        
        def matvec(v):
            θ = v.reshape(shape)
            result = contract_Heff(L_env, R_env, W, θ)
            return result.ravel()
        
        dim = np.prod(shape)
        H_eff = LinearOperator((dim, dim), matvec=matvec, dtype=dtype)
        E, v = eigsh(H_eff, k=1, v0=θ_init.ravel(), which='SA', tol=1e-10)
        θ_opt = v[:, 0].reshape(shape)
      </usage>
    </eigensolver>

    <coarse_eigenvalue title="Coarse-Space Eigenvalue Problem">
      <description>
        Solve the small generalized eigenvalue problem:
        H_coarse c = λ S_coarse c
      </description>
      <implementation>
        from scipy.linalg import eigh
        
        def solve_coarse_space(H_coarse, S_coarse, regularization=1e-10):
            # Regularize S if ill-conditioned
            U, s, Vh = np.linalg.svd(S_coarse)
            s_inv_sqrt = np.where(s > regularization, 1.0 / np.sqrt(s), 0.0)
            S_inv_sqrt = U @ np.diag(s_inv_sqrt) @ Vh
            
            # Transform to standard eigenvalue problem
            H_transformed = S_inv_sqrt @ H_coarse @ S_inv_sqrt.T
            
            # Solve
            eigenvalues, eigenvectors = eigh(H_transformed)
            
            # Transform back
            c_star = S_inv_sqrt.T @ eigenvectors[:, 0]  # Lowest eigenvalue
            
            # Normalize
            c_star = c_star / np.linalg.norm(c_star)
            
            return eigenvalues[0], c_star
      </implementation>
    </coarse_eigenvalue>

    <canonical_forms title="MPS Canonical Forms">
      <left_canonical>
        All tensors except rightmost are left-isometric: Σ_s A^s† A^s = I
        Achieved by: QR decomposition sweeping left to right
      </left_canonical>
      <right_canonical>
        All tensors except leftmost are right-isometric: Σ_s B^s B^{s†} = I
        Achieved by: LQ decomposition sweeping right to left
      </right_canonical>

        Site i is the orthogonality center.
        This is what A2DMRG needs for local micro-steps.
      </i_orthogonal>
    </canonical_forms>
  </numerical_methods>

  <complex_handling>
    <overview>
      For complex wavefunctions (e.g., Josephson junction), special care is needed
      with conjugation operations.
    </overview>
    
    <conjugation_rules>
      <overlap>
        Inner product: ⟨φ|ψ⟩ = φ.H @ ψ (Hermitian conjugate, not just transpose)
        In quimb: overlap = psi_bra.H @ psi_ket
      </overlap>
      
      <environments>
        Environment update formulas use conjugate of ket tensor:
        L_new = contract(L_prev, A, W, A.conj())  # Note: A.conj(), not A.T
        R_new = contract(R_next, B, W, B.conj())
      </environments>
      
      <coarse_matrices>
        H_coarse and S_coarse must use Hermitian inner products:
        H_coarse[i,j] = ⟨Y^(i), H Y^(j)⟩ = (Y^(i))† H Y^(j)
        S_coarse[i,j] = ⟨Y^(i), Y^(j)⟩ = (Y^(i))† Y^(j)
      </coarse_matrices>
    </conjugation_rules>
    
    <dtype_handling>
      Always propagate dtype consistently:
      - If MPS is complex128, environments must be complex128
      - If MPS is float64, can use float64 everywhere
      - Check with: mps.dtype, tensor.dtype
    </dtype_handling>
  </complex_handling>

  <mpi_communication>
    <pattern>
      A2DMRG has a different communication pattern than PDMRG:
      - Phase 2 (local micro-steps): NO communication (embarrassingly parallel)
      - Phase 3 (coarse-space): Allreduce to sum H_coarse and S_coarse contributions
      - Phase 4 (compression): Can be done on rank 0, then broadcast, OR distributed
    </pattern>
    
    <site_distribution>
      def distribute_sites(d, n_procs, rank):
          sites_per_proc = d // n_procs
          remainder = d % n_procs
          
          if rank &lt; remainder:
              start = rank * (sites_per_proc + 1)
              count = sites_per_proc + 1
          else:
              start = remainder * (sites_per_proc + 1) + (rank - remainder) * sites_per_proc
              count = sites_per_proc
          
          return list(range(start, start + count))
    </site_distribution>

    <coarse_space_communication>
      # Each processor computes partial H_coarse and S_coarse
      H_local = np.zeros((d+1, d+1), dtype=dtype)
      S_local = np.zeros((d+1, d+1), dtype=dtype)
      
      for i in my_sites:
          for j in range(d+1):
              H_local[i, j] = compute_H_element(Y[i], Y[j])
              S_local[i, j] = compute_overlap(Y[i], Y[j])
      
      # Allreduce to get global matrices
      H_coarse = np.zeros_like(H_local)
      S_coarse = np.zeros_like(S_local)
      comm.Allreduce(H_local, H_coarse, op=MPI.SUM)
      comm.Allreduce(S_local, S_coarse, op=MPI.SUM)
      
      # Rank 0 solves eigenvalue problem
      if rank == 0:
          E, c_star = solve_coarse_space(H_coarse, S_coarse)
      else:
          c_star = np.empty(d+1, dtype=dtype)
      
      # Broadcast solution
      comm.Bcast(c_star, root=0)
    </coarse_space_communication>

    <updated_mps_distribution>
      After coarse-space, need to form linear combination Ỹ = Σⱼ c*ⱼ Y^(j).
      
      Option 1 (Simple): Gather all Y^(j) to rank 0, form combination, compress, broadcast
      Option 2 (Distributed): Each processor forms local contribution, Allreduce
      
      For moderate d (~100), Option 1 is simpler and sufficient.
    </updated_mps_distribution>
  </mpi_communication>

  <common_pitfalls>
    <pitfall name="Using PDMRG components">
      WRONG: Reuse V matrix, neighbor exchange, staggered sweeps from PDMRG
      CORRECT: A2DMRG is a completely different algorithm
      WHY: A2DMRG uses additive Schwarz, not real-space partitioning
    </pitfall>
    
    <pitfall name="Missing coarse-space step">
      WRONG: Just combine local updates without optimization
      CORRECT: Must solve coarse-space eigenvalue problem
      WHY: Optimal linear combination is critical for convergence
    </pitfall>
    
    <pitfall name="Ill-conditioned mass matrix">
      WRONG: Direct inversion of S_coarse
      CORRECT: Use SVD-based pseudoinverse with regularization
      WHY: S_coarse can be nearly singular
    </pitfall>
    
    <pitfall name="Wrong orthogonal form">
      WRONG: Use arbitrary MPS for local micro-steps
      CORRECT: Each site i needs i-orthogonal decomposition
      WHY: Environments are only valid for specific orthogonality centers
    </pitfall>
    
    <pitfall name="Communication during local steps">
      WRONG: Synchronize or exchange data during Phase 2
      CORRECT: Local micro-steps are completely independent
      WHY: Embarrassingly parallel is the whole point of A2DMRG
    </pitfall>
    
    <pitfall name="Forgetting to include original state">
      WRONG: Only combine updated states Y^(1), ..., Y^(d)
      CORRECT: Include Y^(0) = original state in coarse space
      WHY: Original state is part of the coarse-space basis
    </pitfall>
    
    <pitfall name="Two-site vs one-site indexing">
      WRONG: Same loop for both one-site and two-site
      CORRECT: Two-site has d-1 micro-steps (bonds), one-site has d (sites)
      WHY: Two-site works on bonds, not sites
    </pitfall>
    
    <pitfall name="Tensor shape confusion with quimb">
      WRONG: Assume quimb uses (left, phys, right) ordering
      CORRECT: Always check tensor.inds and transpose if needed
      WHY: quimb may use different internal conventions
    </pitfall>
    
    <pitfall name="Forgetting complex conjugation">
      WRONG: Use A.T instead of A.conj() for complex wavefunctions
      CORRECT: Inner products and environment updates use Hermitian conjugate
      WHY: Quantum mechanics requires Hermitian conjugate, not transpose
    </pitfall>
    
    <pitfall name="Not matching serial DMRG precision">
      WRONG: Accept small deviations from serial DMRG
      CORRECT: Results must match to machine precision (1e-10)
      WHY: This validates that the parallel algorithm is correct
    </pitfall>
  </common_pitfalls>
